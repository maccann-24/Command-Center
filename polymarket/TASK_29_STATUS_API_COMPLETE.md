## Implementation Details

### System Status Logic

**Determines if trading system is running or stalled:**

```python
def get_system_status() -> str:
    # Query most recent event from event_log
    last_event = get_latest_event()
    
    # Check if within last 5 minutes
    if last_event.timestamp > (now - 5 minutes):
        return "running"
    else:
        return "stalled"
```

**Status values:**
- `"running"` — Recent activity (event within 5 minutes)
- `"stalled"` — No recent activity (no events for >5 minutes)
- `"unknown"` — No events in database yet

### Graceful Degradation

**All database queries are wrapped in try-except:**

```python
def get_portfolio_data() -> Dict[str, Any]:
    try:
        portfolio = get_portfolio()
        return portfolio.to_dict()
    except Exception as e:
        # Return default values if DB unavailable
        return {
            "cash": 0.0,
            "total_value": 0.0,
            "note": "Database unavailable"
        }
```

**Why?**
- API remains responsive even if DB down
- Returns sensible defaults
- Includes "note" field to indicate fallback

## Production Deployment

### 1. Run as Background Service

**Using systemd:**

Create `/etc/systemd/system/polymarket-api.service`:
```ini
[Unit]
Description=Polymarket Trading Bot API
After=network.target

[Service]
Type=simple
User=ubuntu
WorkingDirectory=/home/ubuntu/polymarket
EnvironmentFile=/home/ubuntu/polymarket/.env
ExecStart=/home/ubuntu/polymarket/.venv/bin/uvicorn api.endpoints:app --host 0.0.0.0 --port 8000
Restart=on-failure

[Install]
WantedBy=multi-user.target
```


## Design Decisions

### 1. Why FastAPI?

- **Async support** — Non-blocking I/O for concurrent requests
- **Auto-documentation** — Swagger UI generated automatically
- **Type hints** — Request/response validation
- **Performance** — Faster than Flask/Django
- **Modern** — WebSocket support, dependency injection

### 2. Why separate endpoints?

- **Efficiency** — `/portfolio` is faster than full `/status`
- **Granularity** — Fetch only what you need
- **Caching** — Can cache different endpoints independently
- **Monitoring** — Different poll rates per endpoint

### 3. Why 5-minute threshold?

- **Trading cycle** — System runs 60s cycles by default
- **Tolerance** — Allows for occasional delays
- **Alert balance** — Not too sensitive, not too slow
- **Configurable** — Can be changed per deployment
